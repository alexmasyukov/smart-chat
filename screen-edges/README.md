# screen-edges — детектор линий на экране

Находит **длинные яркие грани** (горизонтальные «полки» и вертикальные «стены»)
на экране раз в 2 секунды и отдаёт их по HTTP. Прозрачный оверлей рисует их
зелёными линиями поверх рабочего стола. Задача — дать «карту опор», по которой
десктоп-кот (`../mascot/`) будет ходить и лазать, как настоящий кот по полкам и
шкафам.

Без LLM — чистое компьютерное зрение (OpenCV).

## Как работает

Подход — **морфология** (надёжна на интерфейсах: панели, вкладки, карточки, меню).
Ищем только горизонталь и вертикаль — «сетку интерфейса», без диагоналей.

```
screencapture → grayscale → adaptive threshold → морф-открытие H/V ядром →
   компоненты связности → фильтры(длина≥15%, вытянутость, однородность вдоль,
   контраст поперёк) → слияние коллинеарных → /edges (JSON) → Swift-оверлей
```

- **Захват** — нативный `screencapture -x` (основной дисплей, без доп. зависимостей).
- **Бинаризация** — adaptive threshold ОБЕИХ полярностей (и тёмные линии на
  светлом фоне, и светлые на тёмном) → работает и на светлой, и на тёмной теме.
- **Морфология** — открытие длинным горизонтальным / вертикальным ядром оставляет
  только протяжённые H/V структуры; всё непротяжённое стирается.
- **Отсев текста** — морфология сама по себе обводит каждую строку текста. Ключевой
  фильтр: у сплошной линии яркость ВДОЛЬ однородна (низкий std ~0–18), у строки
  текста скачет (std ~30–75). Порог `SE_ALONG_STD` отделяет линии от текста.
- **Контраст ПОПЕРЁК** = `max(step, ridge)`: step — перепад на границе, ridge —
  насколько сама линия ярче/темнее фона (ловит и грани, и тонкие яркие полоски).
- **Слияние** — коллинеарные куски одной линии сливаются в цельный отрезок.
- **Координаты — нормализованные** (0..1, y сверху вниз). Потребитель масштабирует
  под свои point-размеры окна, без возни с retina/backingScaleFactor.

Замер: весь цикл (захват + морфология + фильтры) ~0.2с, сама детекция ~15–20мс.
При интервале 2с это ~10% бюджета.

## Запуск

```bash
./run.sh
```

Соберёт оверлей (если нужно), поднимет детектор на `:8130` (если не запущен) и
запустит прозрачный оверлей. `Ctrl+C` закрывает оверлей; детектор остаётся жить
в фоне (его же читает кот). Остановить детектор:

```bash
pkill -f 'screen-edges/detect.py'
```

## HTTP API

```bash
curl -s http://127.0.0.1:8130/edges | python3 -m json.tool
curl -s http://127.0.0.1:8130/health
```

`/edges` → JSON:

```json
{
  "ts": 1752300000.0, "w": 3456, "h": 2234, "count": 8, "ms": 245.0,
  "segments": [
    {"x1":0.004,"y1":0.031,"x2":0.996,"y2":0.031,"o":"H","len":0.992,"contrast":42.5}
  ]
}
```

- `x1,y1,x2,y2` — концы отрезка, нормализованные 0..1 (y сверху вниз).
- `o` — `"H"` (горизонталь / полка), `"V"` (вертикаль / стена), `"D"` (диагональ,
  если включена).
- `len` — нормализованная длина; `contrast` — заметность грани (0..255).
- `w,h` — пиксели снятого экрана; `ms` — время цикла; `ts` — время замера (Unix).

Сегменты отсортированы: длинные и контрастные первыми.

## Отладка без оверлея

```bash
python3 detect.py --snapshot   # снимет экран и сохранит out/preview.png с линиями
```

## Тюнинг (переменные окружения)

| Переменная           | По умолч. | Что                                              |
|----------------------|-----------|--------------------------------------------------|
| `SE_PORT`            | `8130`    | порт HTTP                                         |
| `SE_INTERVAL`        | `2.0`     | период съёмки, сек                                |
| `SE_WORK_WIDTH`      | `1600`    | рабочая ширина кадра (меньше = быстрее/грубее)    |
| `SE_MIN_LEN_FRAC`    | `0.40`    | мин. длина линии как доля стороны экрана          |
| `SE_MIN_CONTRAST`    | `8`       | мин. контраст поперёк (0..255); выше = строже      |
| `SE_ALONG_STD`       | `22`      | макс. разброс вдоль линии; ниже = строже к тексту |
| `SE_MIN_ASPECT`      | `3`       | вытянутость компонента (длина/толщина)            |
| `SE_ADAPT_BLOCK`     | `15`      | окно adaptive threshold (нечётное)                |
| `SE_MORPH_FRAC`      | `0.05`    | длина морф-ядра как доля стороны                  |

Оверлей: `EDGES_URL` (адрес `/edges`), `POLL` (период опроса, сек).

## Требования

- Python 3 с `opencv-python`, `numpy` (уже стоят в системном `python3`).
- **Screen Recording** для терминала, из которого запускается детектор
  (Системные настройки → Конфиденциальность → Запись экрана). Без него снимок
  выходит чёрным — в логе будет предупреждение.
- Оверлей — только основной дисплей (мультимонитор пока не поддержан).

## Важно: петля обратной связи

Окно оверлея скрыто от захвата экрана (`NSWindow.sharingType = .none`). Без
этого `screencapture` снимал бы экран вместе с нарисованными зелёными линиями,
детектор переобнаруживал бы их как яркие грани и рисовал снова — линии
«застревали» бы навсегда, даже когда исходная грань UI исчезла.
