# ft — дообучение классификатора (LoRA на LFM2.5-350M)

Дообучаем маленькую модель **LFM2.5-350M** под конкретный набор инструментов,
чтобы она классифицировала запрос → имя инструмента (или `none`) так же точно,
как 1.2B, но быстрее (~0.14с) и легче (~0.4 ГБ).

> Unsloth здесь не при чём — он требует NVIDIA/CUDA. На Apple Silicon дообучение
> делаем встроенным LoRA-тренером **mlx-lm** (`mlx_lm.lora`).

## Как переобучить под новый набор инструментов

1. Правишь инструменты в `../classifier.py` (`TOOLS`) — это источник истины для имён.
2. Правишь шаблоны формулировок в `gen_dataset.py` (`TOOLS_PHRASES`, `OPEN_*`, `NONE`).
3. Запускаешь весь пайплайн:
   ```bash
   cd mlx-chat && ./ft/train.sh
   ```
   Это: генерация датасета → LoRA (~2 мин) → fuse → eval на held-out.
4. `classifier.py` по умолчанию берёт `ft/fused` — новая модель сразу в деле.

## Результаты (тестовый набор из 6 инструментов + none)

| | базовая 350M | 350M + LoRA | 1.2B Instruct |
|---|---|---|---|
| Held-out (17 фраз) | 4/17 | **17/17** | 17/17 |
| Новые формулировки | — | **12/12** | — |
| Скорость / размер | 0.13с / 0.4ГБ | **0.14с / 0.4ГБ** | 0.33с / 1.3ГБ |

## Файлы

- `gen_dataset.py` — генератор `data/{train,valid}.jsonl` (chat-формат) из шаблонов.
- `eval.py` — точность на held-out фразах (короткий промпт + грамматика-enum).
- `train.sh` — весь пайплайн одной командой.
- `data/` — сгенерированный датасет (можно коммитить, мелкий).
- `adapters/`, `fused/` — веса LoRA и сплавленная модель (в git не идут, ~0.4ГБ).

## Ключевые детали

- **Короткий системный промпт** и в обучении, и в инференсе (few-shot не нужен —
  маппинг зашит обучением). Он один и тот же в `gen_dataset.py` и `classifier.py`.
- **Грамматика-enum (Outlines) остаётся** и после дообучения: обучение улучшает
  *выбор*, Outlines гарантирует *формат*. Складываются.
- Гиперпараметры (в `train.sh`): 500 итераций, batch 8, 8 слоёв, lr 1e-4. Val loss
  сходится к ~0.24 уже к 100-й итерации.
- Held-out фразы (`eval.py: CASES`) исключаются из train автоматически.
