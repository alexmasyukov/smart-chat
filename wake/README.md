# wake — детектор пробуждающего слова «Эй, кот»

Локальный wake word detector: микрофон → признаки → маленькая нейросеть → «фраза
произнесена / нет». Всё на маке (Apple Silicon), без облака, приватно, на CPU.

Слушает поток с микрофона и, когда слышит **«Эй, кот»**, печатает `✅ РАСПОЗНАЛ`.
Задача — *keyword spotting (KWS)*. Это **детектор фразы**: срабатывает на «Эй, кот»
от любого голоса, а не только владельца.

📄 **[TRAINING.md](TRAINING.md)** — подробный разбор: как готовили данные, как обучали,
на какие грабли наступили, метрики. Читать, если нужно разобраться или воспроизвести.

## Архитектура

Рецепт по разбору [hey-gaia](https://aryanranderiya.com/agent-convos/hey-gaia-wake-word-trained/):

```
аудио 16кГц → melspectrogram(ONNX) → embedding(ONNX, Google speech) → (16,96) → Conv1D-голова → sigmoid
```

- **Признаки** — предобученные модели openWakeWord (melspec + Google speech embedding),
  «заморожены». Окно 2 сек → тензор `(16 кадров, 96)`.
- **Голова** — обучаемый Conv1D (свёртка по времени ловит паттерн «эй»→«кот»),
  ~30K параметров, экспорт в ONNX (~200 КБ).

### ⚠️ Главный урок: обучать ТАМ ЖЕ, где инференс

melspectrogram-ONNX даёт **разные** признаки на x86 (Colab) и ARM (M-чип) — corr ~0.79
для одного и того же файла (различие в FFT между архитектурами). Модель, обученную на
Colab-признаках, локально запустить нельзя (скор 0). Поэтому **и признаки, и обучение —
на маке**. Обучение крохотное (Conv1D-голова), идёт локально за минуты; GPU/Colab не нужны.

## Данные

| Класс | Источник | Кол-во |
|---|---|---|
| Позитивы | ElevenLabs «Эй, кот», ~70 русских голосов × варианты | ~1000 клипов |
| Негатив: реальная речь | Golos (Сбер, русская речь) — **главный убийца ложняков** | ~20k окон |
| Негатив: похожие слова | ElevenLabs «эй код / кот / эй кто…» (вес ×4) | ~500 клипов |
| Негатив: шум | синтетика (белый/розовый/тоны/чирпы/клики) | ~2200 окон |
| Негатив: эфир | запись реального фона микрофона | ~285 окон |

Позитивы аугментируются: темп/высота ±10%, реверберация (комната/микрофон), сдвиг
позиции в окне, шум, громкость. Обучение — взвешенный BCE (баланс классов + ×4 на
похожие слова) + SpecAugment (маскирование кадров/каналов) + cosine LR.

## Файлы

| Файл | Что делает |
|---|---|
| `detect.py` | **живой детектор**: микрофон → onnx → порог. Энергетический гейт + сглаживание |
| `wake_common.py` | общие функции признаков (аугментация, окна, эмбеддинг) |
| `build_cache.py` | **шаг 1**: считает эмбеддинги всех групп → кэш `out/cache/feat_*.npy` (долго) |
| `train_local.py` | **шаг 2**: грузит кэш → Conv1D (MPS) → экспорт `kot_slushai.onnx` (~30с) |
| `embed_speech.py` | скачивает русский корпус (Golos) и считает фичи-негативы → `out/neg_speech_feats.npy` |
| `gen_dataset.py` | генерация позитивов «Эй, кот» через ElevenLabs (ключ в `.env`) |
| `gen_negatives.py` | генерация речевых + похожих негативов через ElevenLabs |
| `test_model.py` / `record_test.py` | запись и прогон тест-набора своим голосом (held-out) |
| `kot_slushai.onnx` | обученная модель |

**Важно:** дорогой пересчёт признаков (`build_cache.py`) отделён от обучения
(`train_local.py`). Признаки считаются 1 раз в кэш; дальше обучение головы —
секунды, можно свободно крутить архитектуру/веса/порог без пересчёта.

## Запуск детектора

```bash
.venv/bin/python detect.py
# крутилки:
THRESHOLD=0.92 .venv/bin/python detect.py   # порог срабатывания (0..1)
GATE=1500 .venv/bin/python detect.py         # порог тишины (пиковая амплитуда)
CONSEC=3 .venv/bin/python detect.py          # окон подряд выше порога (сглаживание)
```

Молчание → `скор 0.00`. «Эй, кот» → `✅ РАСПОЗНАЛ`. Обработка окна ~45 мс, реакция ~0.3–0.5 с.

## Переобучение

```bash
# (опц.) больше данных
VOICES=70 .venv/bin/python gen_dataset.py     # позитивы «Эй, кот»
VOICES=12 .venv/bin/python gen_negatives.py   # чужая речь + похожие слова
TARGET=25000 .venv/bin/python embed_speech.py # реальная речь (Golos) в негативы

# шаг 1: построить кэш признаков (долго, ~10 мин; при смене данных)
.venv/bin/python build_cache.py
# шаг 2: обучить голову из кэша (быстро, ~30с; крути сколько хочешь)
.venv/bin/python train_local.py
```

## Метрики (honest held-out: 150 невиданных голосов + отложенная реальная речь Golos)

| порог | recall (невиданные голоса) | FP похожие слова | FP реальная речь |
|---|---|---|---|
| 0.70 | 98.8% | 0.0% | 0.07% |
| 0.85 | 98.5% | 0.0% | 0.07% |
| 0.90 | 98.0% | 0.0% | 0.07% |

Проверка через реальный путь детекта (`_get_embeddings`→onnx): позитивы 14/14 = 1.00.

Обучающие данные: ~22.5k позитивных окон (эй кот, 60 голосов × аугментация),
~8.5k реальной русской речи (Golos), похожие слова ×4, шум/эфир.
Эталон hey-gaia: 99% / 0.4%.

### ⚠️ Критично: признаки должны быть ДЕТЕРМИНИРОВАНЫ
`F.embed_clips` (батч openWakeWord) из-за `np.empty` возвращает РАЗНЫЕ признаки
при каждом вызове для одного аудио → обучение и детект на случайных фичах, всё
разваливается. Используем только `F._get_embeddings` (потоковый, детерминирован);
`embed_all` в `wake_common.py` собран на нём (мелспек по клипу + один эмбеддинг-вызов).

## Статус и что дальше

Recall на фразе решён (~99%). Основной фронт — снижение ложных срабатываний на
реальной речи: рычаг — **ещё больше реальной речи в негативах** (эталон: сотни тысяч
окон), плюс сглаживание на инференсе (`CONSEC`) и подбор порога.
