# Assistant — выводы по реализации (плавность 120 Гц, окно, эффект)

Итог сессии: как добились плавного голос-реактивного свечения вокруг выреза под
камеру (notch) на 120 Гц без пожирания CPU. Основной код — `main.swift`.
Песочница других анимаций и эталон fps — `experiments.swift` (в проде не запускается).

## Что рисуем

Свечение, **оборачивающее вырез**: конический радужный градиент вращается ПОД
статичной маской-формой (размытая обводка контура выреза). Цвета переливаются
вдоль контура, форма стоит на месте. Голос раздувает свечение наружу и ярчит.

## Ключевые выводы по производительности

1. **Metal здесь — ловушка.** `CAMetalLayer` — swapchain-поверхность, которую
   WindowServer пересобирает КАЖДЫЙ vsync (120 Гц) даже без презентов и при
   `isPaused=true`. На прозрачном окне это ~30% CPU в WindowServer **постоянно**,
   плюс рывки. Metal хорош для непрозрачного фуллскрина (игры), не для прозрачного
   оверлея. Замеры: прозрачное окно с Metal +30% WindowServer в покое; статичное
   Core Animation окно того же размера +1–2%.

2. **Гнать анимацию через `CADisplayLink`, а не `Timer`.** `Timer` не привязан к
   vsync — его тики коалесцируются и «плавают», из-за чего движение рвётся, хотя
   fps честные 120. `CADisplayLink` (макрос `displayLink(target:selector:)`,
   macOS 14+, `preferredFrameRateRange` до 120) даёт кадры ровно по 8.33 мс,
   джиттер ~0. Это проверено: и обычное окно, и прозрачный оверлей держат
   стабильные 120.0 fps через CADisplayLink.

3. **«120 fps, но дёргается» = смотреть на частоту ДАННЫХ, а не кадров.** Микрофон
   отдаёт уровень ~47–94 раза в секунду (раз на аудио-буфер), а рисуем 120/с.
   Если применять уровень напрямую — размер держится константой 2–3 кадра и
   прыгает (ступеньки). Решение: **интерполировать значение на каждом кадре**
   (`dispLevel += (target - dispLevel) * k`, быстрый attack ~0.28 / плавный
   release ~0.16) + буфер микрофона поменьше (512). Рост становится плавным.

4. **Тяжёлое считать один раз (запекать в текстуру).** `CALayer.filters`
   (CIGaussianBlur) в рантайме каждый кадр — худший враг (offscreen-проход,
   пересчёт на CPU): и рывки, и обрезание. Маска (`mask`, `cornerRadius+
   masksToBounds`) — тоже offscreen-проход, дорожает с площадью → просадка при
   увеличении. Поэтому:
   - размытая обводка/мягкие края **запекаются в CGImage один раз** (гаусс через
     Core Image единожды при старте);
   - в рантайме — только `transform` (scale/rotate) и `opacity` готовых текстур.
   Это дёшево и держит 120 даже при масштабировании (проверено на орбе и кольцах:
   с маской теряли fps на scale-up, после запекания — держат).

5. **Перелив требует компромисса.** Вращать ЗАПЕЧЁННУЮ текстуру-обёртку нельзя —
   исказит форму выреза. Поэтому форму держим статичной маской, а радугу вращаем
   слоем под ней (`conic.transform` rotation в tick). Это возвращает рантайм-
   маскирование (offscreen-проход) → WindowServer нагружается, НО на маленькой
   площади обводки выреза fps держит 120. Приемлемая цена за перелив.

6. **Сон в тишине.** Когда голос стих (`holdTime` ~0.7 с) — `CADisplayLink`
   останавливается, кадр статичный, WindowServer кеширует → ~0 overhead. Голос
   будит из аудио-потока (`mic.onVoice` → `DispatchQueue.main.async { wake() }`).

## Как построено окно

- borderless `NSWindow`, `isOpaque=false`, `backgroundColor=.clear`,
  `hasShadow=false` — прозрачный оверлей.
- `level = CGShieldingWindowLevel()` — выше строки меню и выреза, рисуем прямо в
  зоне notch. (Уровень/зона/размер на нагрузку почти не влияют — влияет только
  сам факт непрерывной анимации прозрачного контента.)
- `ignoresMouseEvents=true` — click-through, не мешает работе.
- `collectionBehavior=[.canJoinAllSpaces, .fullScreenAuxiliary, .stationary]`.
- `setActivationPolicy(.accessory)` + `LSUIElement` — без иконки в доке.
- Геометрия выреза: `NSScreen.safeAreaInsets.top` (высота notch),
  `auxiliaryTopLeftArea.maxX .. auxiliaryTopRightArea.minX` (ширина). Окно —
  вокруг выреза с запасом `mx/my` под разрастание свечения (иначе обрежется при
  scale). Микрофон требует .app-бандл с `NSMicrophoneUsageDescription` (TCC).

## Эталон для проверки (в experiments.swift)

Чёткий кружок, крутящийся с постоянной скоростью (на голос не реагирует) — по
нему на глаз видно, дёргается ли картинка. Плюс счётчик fps/джиттера через
`CADisplayLink.timestamp`.
