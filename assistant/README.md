# assistant

Нативное macOS-приложение (Swift + AppKit): переливающееся голос-реактивное
свечение, **оборачивающее вырез под камеру (notch)** — в стиле Siri. Формы/окна
нет: прозрачный click-through оверлей поверх всего. Пока тихо — свечение облегает
вырез статично; когда говоришь — переливается цветами и разрастается наружу по
громкости голоса. Работает плавно на 120 Гц (ProMotion).

## Файлы

- **`main.swift`** — боевой эффект: обёртка выреза. Радужный конический градиент
  вращается ПОД статичной запечённой маской-обводкой выреза (перелив цветов, форма
  на месте); голос раздувает свечение наружу и ярчит.
- **`tuner.swift`** → `Tuner.app` — форма с ползунками для ЖИВОЙ настройки эффекта
  (16 параметров: размер, толщина ореола, размытие, скругление, спад к краю,
  насыщенность, яркость, скорость вращения, размах/attack/release/gamma/
  чувствительность, прозрачность). Именованные шаблоны (сохранить/выбрать/удалить),
  хранятся в `~/Library/Application Support/AssistantTuner/presets.json`. Внизу —
  копируемый дамп всех значений. Не трогает `main.swift`.
- **`experiments.swift`** — АРХИВ: песочница других анимаций (эквалайзер-бары,
  пульсирующие кольца, эталон-спиннер + счётчик fps). В проде не запускается.
- **`NOTES.md`** — технические выводы: почему CADisplayLink (не Timer/Metal), как
  держим 120 Гц, запекание текстур вместо рантайм-масок/фильтров, сглаживание
  уровня по кадрам, сон в тишине, как построено прозрачное окно.
- **`build.sh` / `run.sh`** — сборка/запуск боевого приложения (`Assistant.app`).

## Запуск

```bash
cd assistant
./run.sh                 # собрать (если надо) и запустить с логами; Ctrl+C — закрыть
./run.sh open            # запустить как приложение (open Assistant.app)

# тюнер:
swiftc -O tuner.swift -o Tuner.app/Contents/MacOS/Tuner \
  -framework AppKit -framework AVFoundation -framework QuartzCore
./Tuner.app/Contents/MacOS/Tuner
```

`.app`-бандл нужен для доступа к микрофону (TCC, `NSMicrophoneUsageDescription`).

## Стек и ключевые решения

- **Core Animation, НЕ Metal.** `CAMetalLayer` на прозрачном окне пересобирается
  WindowServer'ом каждый vsync → жрёт CPU постоянно. Здесь весь эффект — CA-слои.
- **`CADisplayLink` (не `Timer`)** — vsync-locked 120 Гц без джиттера.
- **Запекать тяжёлое один раз** (размытие/обводку в CGImage), в рантайме — только
  `transform`/`opacity` готовых текстур (маски/CIFilter каждый кадр = просадка).
- **Сглаживание уровня по кадрам** — микрофон обновляется реже рендера; интерполяция
  на каждом кадре убирает ступеньки роста.
- Прозрачный borderless оверлей, `CGShieldingWindowLevel` (поверх строки меню и
  выреза), `ignoresMouseEvents`, `.accessory` (без иконки в доке).

Подробности — в `NOTES.md`.

## Статус

Боевой эффект (`main.swift`) готов и одобрен. Тюнер (`tuner.swift`) — для подбора
финальных настроек; когда выбран нужный шаблон/дамп, значения вкатываются в
`main.swift`. Разработка тюнера — в ветке `assistant-tuner`.
