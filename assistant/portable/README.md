# NotchGlow — переносимый модуль свечения вокруг выреза

Готовый эффект из проекта `assistant`, вынесенный для использования в другом
приложении. Без окна настроек, вся настройка — через конфиг.

## Что копировать в другой проект

| Файл | Обязателен | Зачем |
|---|---|---|
| `NotchGlow.swift` | **да** | весь эффект: окно-оверлей, микрофон, рендер |
| `notch-glow.json` | нет | конфиг с дефолтными значениями (можно и в коде) |

Больше ничего не нужно — модуль self-contained, зависит только от системных
фреймворков: **AppKit, AVFoundation, QuartzCore, CoreImage**.

## Требования принимающего проекта

- macOS 14+ (`CADisplayLink` на `NSView`).
- Приложение — `.app`-бандл, в `Info.plist`:
  `NSMicrophoneUsageDescription` (иначе система не даст микрофон).
  Не нужно, если подаёшь уровень сам (`useBuiltInMic: false`).
- Линковка: `-framework AppKit -framework AVFoundation -framework QuartzCore
  -framework CoreImage` (в Xcode-проекте они и так есть).

## Использование

```swift
// 1) Просто запустить с дефолтом
let glow = NotchGlow()
glow.start()          // из главного потока, после старта NSApplication

// 2) Конфиг из JSON-файла в бандле
let cfg = NotchGlowConfig.load(
    Bundle.main.url(forResource: "notch-glow", withExtension: "json"))
let glow = NotchGlow(config: cfg)
glow.start()

// 3) Конфиг в коде
var cfg = NotchGlowConfig()
cfg.voiceScale = 0.9
cfg.rotSec = 6
let glow = NotchGlow(config: cfg)
glow.start()

// Смена настроек на лету (перезапекает маску — не звать каждый кадр)
glow.apply(cfg)

// Свой источник звука вместо встроенного микрофона
var cfg = NotchGlowConfig(); cfg.useBuiltInMic = false
let glow = NotchGlow(config: cfg); glow.start()
glow.setLevel(0.42)   // 0…1, зови из своего аудио-пайплайна

// Убрать
glow.stop()

// Сохранить подобранный пресет
print(glow.config.jsonString())
```

Модуль сам находит экран с вырезом, ставит прозрачное click-through окно поверх
всего (включая меню-бар) и не крадёт фокус. Никакого UI не добавляет.

## Параметры конфига

Любое поле в JSON можно опустить — подставится дефолт.

**Форма и ореол** (смена → перезапекание маски, дороже):
- `outlineWidth` (64) — толщина ореола, px
- `blur` (5) — размытие маски, px
- `corner` (0.7) — скругление нижних углов выреза, доля
- `falloff` (1.5) — спад яркости наружу (больше = ореол прижат к кромке)
- `saturation` (1.4) — насыщенность палитры
- `brightness` (1.0) — яркость палитры

**Размер и движение** (дёшево, применяется в кадре):
- `scaleX` / `scaleY` (1.0) — базовый масштаб обёртки
- `rotSec` (9) — период оборота перелива цветов, сек
- `voiceScale` (0.7) — насколько раздувается на полном голосе
- `idleOpacity` (0.85) — непрозрачность в покое
- `voiceOpacity` (0.15) — прибавка непрозрачности на голосе

**Реакция на звук:**
- `attack` (0.28) — скорость роста на кадр (больше = резче прыгает)
- `release` (0.16) — скорость затухания на кадр
- `gamma` (1.1) — контраст реакции
- `sensitivity` (1.0) — чувствительность микрофона

**Интеграция:**
- `useBuiltInMic` (true) — слушать микрофон самому; false → `setLevel(_:)`
- `holdTime` (0.7) — секунд тишины до засыпания анимации
- `marginX` (140) / `marginY` (120) — запас окна вокруг выреза; должен вмещать
  ореол при максимальном раздувании, иначе свечение обрежется по краю окна

## Подбор значений

Крутить ползунки удобно в исходном проекте: `assistant/Tuner.app` — живой
предпросмотр всех этих параметров + дамп значений. Подобранные числа
переносятся в `notch-glow.json` один в один (имена совпадают).

## Производительность

- Всё на **Core Animation, не Metal**: `CAMetalLayer` на прозрачном окне
  заставляет WindowServer пересобирать поверхность каждый vsync → постоянный
  расход CPU.
- Маска-обводка с размытием **запекается один раз** в `CGImage`; в рантайме
  только `transform`/`opacity`.
- Анимация идёт через `CADisplayLink` (vsync-locked 120 Гц, без джиттера),
  а в тишине **останавливается** — статичный кадр, накладных расходов ~0.
